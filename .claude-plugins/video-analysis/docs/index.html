Claude Code plugin

video-analysis

A four-skill pipeline for downloading, transcribing, extracting frames from, and analyzing social media video content across every major platform.

/video-download /video-transcribe /video-frames /video-dashboard

The pipeline

Four stages, each a standalone slash command. Run them in sequence or use any stage independently.

Skills

Each skill is a standalone slash command. Use them in order for the full pipeline, or individually as needed.

/video-download

Download videos from any major social media platform using yt-dlp. Falls back to Playwright browser automation when direct download fails, handling login walls and dynamic page rendering.

Twitter/X TikTok YouTube Instagram Facebook

Usage

/video-download

Output

  • MP4 files in the configured output directory
  • JSON metadata sidecar per video
  • Download log with success/failure status

/video-transcribe

Batch transcribe all downloaded videos using OpenAI Whisper. Auto-detects available VRAM and selects the best model (tiny through large-v3). Runs on GPU when available, falls back to CPU.

Usage

/video-transcribe

Output

  • Plain text transcript per video (.txt)
  • Timestamped SRT subtitle file (.srt)
  • Whisper model selection and timing log

/video-frames

Extract frames at configurable intervals (default: every 5 seconds), assemble them into 3x3 grid composites, and run Claude vision analysis on each grid to catalog on-screen text, graphics, and visual elements.

Usage

/video-frames

Output

  • Individual frame images (PNG)
  • 3x3 grid composite images
  • JSON vision analysis per grid (text, objects, scenes)

/video-dashboard

Run keyword-based topic and sentiment analysis across all transcripts and vision data, then generate a self-contained interactive Chart.js dashboard with filtering, sorting, and drill-down.

Usage

/video-dashboard

Output

  • Interactive HTML dashboard (Chart.js)
  • Topic frequency and sentiment charts
  • Per-video detail view with transcript excerpts
  • Exportable CSV data tables

Example workflow

A start-to-finish walkthrough: downloading a public figure's social media videos and producing an analysis dashboard.

1

/video-download

Start the download skill. It prompts for a platform, search query or URL list, and output directory. yt-dlp handles the download; if it fails (login wall, age gate), Playwright launches a headless browser as fallback. 

# Claude prompts you interactively:
> Platform? Twitter/X
> Search query or URL file? urls.txt
> Output directory? ./downloads

# Download begins
[yt-dlp] Downloading 1 of 24: @user/status/1234567890
[yt-dlp] Downloading 2 of 24: @user/status/1234567891
...
[done] 22 succeeded, 2 failed (retrying with Playwright)
[playwright] Retrying 2 failed downloads...
[done] 24/24 videos downloaded to ./downloads
2

/video-transcribe

Point the transcription skill at your downloads directory. It detects available VRAM (e.g., 16 GB on an RTX 4080 Super), picks the best Whisper model, and processes all videos in batch. 

# Auto-detection
[gpu] NVIDIA RTX 4080 Super -- 16 GB VRAM
[whisper] Selected model: large-v3 (best accuracy for 16 GB)

# Batch transcription
[1/24] video_001.mp4 -- 2:34 -- transcribed in 8.2s
[2/24] video_002.mp4 -- 1:12 -- transcribed in 4.1s
...
[done] 24 transcripts written to ./downloads/transcripts/
3

/video-frames

Extract frames every 5 seconds, build 3x3 composite grids, and run Claude vision analysis on each grid. The result is a structured catalog of every visual element that appears on screen. 

# Frame extraction
[ffmpeg] Extracting frames at 5s intervals...
[1/24] video_001.mp4 -- 31 frames extracted
[1/24] Compositing 4 grids (3x3)...

# Vision analysis
[vision] Analyzing grid 1/4 for video_001...
  - On-screen text: "Breaking: Senate votes 52-48"
  - Graphics: lower-third chyron, network logo
  - Scene: press conference, podium, microphones
...
[done] 24 videos analyzed -- results in ./downloads/frames/
4

/video-dashboard

The final stage reads all transcripts and vision analysis, runs keyword-based topic and sentiment scoring, and generates a self-contained HTML dashboard with interactive Chart.js visualizations. 

# Analysis
[analysis] Processing 24 transcripts + vision data...
[topics] Found 8 topic clusters
[sentiment] Scored 24 videos (positive: 12, neutral: 8, negative: 4)

# Dashboard generation
[dashboard] Writing interactive HTML dashboard...
[done] Dashboard saved to ./downloads/dashboard.html
[done] CSV export saved to ./downloads/dashboard-data.csv

Requirements

System dependencies needed for the full pipeline. GPU is optional but significantly accelerates transcription.

Dependency Purpose Status
yt-dlp Video downloading from social media platforms Required
ffmpeg Frame extraction, audio conversion, video processing Required
openai-whisper Speech-to-text transcription (Python package) Required
Pillow Image compositing for 3x3 frame grids (Python package) Required
Playwright Browser automation fallback for download failures Required
NVIDIA GPU + CUDA GPU-accelerated Whisper transcription (10-50x faster) Optional
